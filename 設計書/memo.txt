勤怠管理システムを開発したいです。

##1アーキテクチャ 
フロント：Vue.js
バック：Python(Django)
DB：GCP(MySQL)

##2画面 
1,ログイン画面
パスワードは不要です。オートコンプリート機能を無効
5桁の社員番号です。認証周りの仕様について簡単にしてほしい。
employees.EMPLIDに一致するか。
2,勤怠入力画面
カレンダーは外部ライブラリを一切使用せず、Vue 3 のみで実装する。
カレンダーは日曜日始まりで月間表示、かつ月切替ができる、ユーザーがクリックで日付を選択した際、右側の詳細部分に初期値として就業時間「9:00」と終了時間「18:00」が表示されるという流れ。
時刻ピッカーの実装について、入力値の妥当性チェックはある程度（任せます！）で。

APIについて。
提供するAPIエンドポイントは、主に「ログイン（社員番号チェック）」「勤怠データの取得」「勤怠データの登録・更新」といったもの
認証情報の管理は、セッションベース
フロントエンドとバックエンド間の通信に関して、CORSやCSRF対策の具体的な要件は良く分からないのでお任せで。

の2画面の開発をしてください。
ヘッダーには画面名を表示し、ヘッダーの右下に小さくログインしたユーザで社員名と社員番号を表示するようにする。ログアウトボタン。employees.EMPLNM、employees.EMPLID

##6テーブル定義 
CREATE TABLE employees (
EMPLID VARCHAR(5) PRIMARY KEY, --社員番号
EMPLNM VARCHAR(100) --社員名前
);

CREATE TABLE attendance (
ATTEID VARCHAR(5), --社員番号
ATTEDT date, --日付
ATTEST VARCHAR(4), --就業時間
ATTEET VARCHAR(4), --終了時間
FOREIGN KEY (ATTEID) REFERENCES employees(EMPLID)
);






CREATE TABLE vacation_requests (
REQUID VARCHAR(5), --社員番号
REQUDT VARCHAR(8), --日付
REQUSY VARCHAR(20), --休暇種別
REQUKY VARCHAR(1), --休暇フラグ（1:承認中、2:承認済み、3:却下）
FOREIGN KEY (REQUID) REFERENCES employees(EMPLID)
);
